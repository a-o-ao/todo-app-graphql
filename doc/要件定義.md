# TODOアプリ要件定義

## 非機能要件

### インフラストラクチャ

- 実行環境はDockerFileで構成する
- DBはPostgreSQL17.4を使用する
- docker初回起動時にテーブルまで作成する

### バックエンド

- 最新バージョンのNestJSフレームワークを使用する
- 開発言語は最新バージョンのTypescriptとする
- DBへはGraphQLを利用して接続する

### フロントエンド

- 最新バージョンのNext.jsで作成する
- レスポンシブデザイン

## 機能要件

### 画面構成

#### 一覧画面
- 登録したTODOの一覧を表示する
- 画面表示項目
  - 一覧
    - タイトル(リンク)
    - 完了ボタン(チェックアイコン)
  - 登録ボタン
- 表示制御
  - ステータスが`1:完了`しているものは行をグレーの網掛け表示
  - ステータスの昇順、更新日時の降順で表示する。
- アクション
  - 初期表示
    - `todos` クエリを実行し、取得結果を一覧表示する
  - 一覧
    - タイトルを押下すると、選択したアイテムの詳細編集画面がモーダル表示される
    - 完了ボタンを押下すると、以下のアクションが順に実行される
      - `updateTodo` ミューテーションを実行し、ステータスを`1:完了`に更新する
      - エラーが返されていない場合、行をグレーの網掛け表示にする
  - 登録ボタン押下
    - 詳細編集画面がモーダル表示される
#### 詳細編集画面
- 選択したTODOの内容を表示する
- 編集の場合と新規登録の場合がある。IDが指定されている場合は編集とする
- 表示項目
  - タイトル(テキストボックス)
  - 内容(テキストボックス)
  - 編集の場合
    - 更新日時
    - 完了ボタン(チェックアイコン)
    - 削除ボタン(ゴミ箱アイコン)
  - 保存ボタン
- `タイトル`と`内容`は編集可能
- アクション
  - 初期表示
    - 編集の場合は、`todo(id: Int!)` クエリを実行し、表示項目に取得結果を表示する
    - 登録の場合は、表示項目を空で表示する
  - 完了ボタンを押下すると、以下のアクションが順に実行される
    - `updateTodo` ミューテーションを実行し、ステータスを`1:完了`に更新する
    - 詳細編集画面が非表示となり、一覧画面が表示される
    - 一覧画面の表示が更新される
  - 削除ボタンを押下すると、以下のアクションが順に実行される
    - `deleteTodo` ミューテーションを実行する
    - 詳細編集画面が非表示となり、一覧画面が表示される
    - 一覧画面の表示が更新される
  - 保存ボタンを押下すると、以下のアクションが順に実行される
    - 編集の場合は `updateTodo` ミューテーションを、新規登録の場合は `createTodo` ミューテーションを実行する
    - エラーが返されていない場合
      - 詳細編集画面が非表示となり、一覧画面が表示される
      - 一覧画面の表示が更新される
    - エラーが返された場合
      - エラーメッセージをトースト表示する(保存に失敗しました)

### やらないこと(スコープ外)
- ユーザ機能/ログイン機能
- ページング
- エラー処理
- フロントのバリデーションはブラウザの標準機能に任せる

## 基本設計

### テーブル定義

#### TODOテーブル

テーブル名: `todo`  
補足: `status` は `ENUM型 (todo_status)` を使用、`updated_at` はトリガーで自動更新

| No | カラム名 (論理) | カラム名 (物理) | 型 | 制約・初期値・補足 |
|----|------------------|------------------|-----------------------|---------------------------------------------|
| 1  | ID               | id               | `SERIAL`              | 主キー、自動採番 |
| 2  | タイトル         | title            | `VARCHAR(32)`         | NOT NULL |
| 3  | 内容             | contents         | `VARCHAR(256)`        | NULL許可 |
| 4  | ステータス       | status           | `todo_status`         | NOT NULL、DEFAULT `'PENDING'` |
| 5  | 作成日時         | created_at       | `TIMESTAMP`           | DEFAULT `CURRENT_TIMESTAMP` |
| 6  | 更新日時         | updated_at       | `TIMESTAMP`           | DEFAULT `CURRENT_TIMESTAMP`、更新時にトリガーで更新 |

---

#### ENUM定義

- PENDING: タスクは未完了状態です。
- COMPLETED: タスクは完了しています。

```sql
CREATE TYPE todo_status AS ENUM ('PENDING', 'COMPLETED');
```

#### 更新日時トリガー関数とトリガー

```sql
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
   NEW.updated_at = CURRENT_TIMESTAMP;
   RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_update_timestamp
BEFORE UPDATE ON todo
FOR EACH ROW
EXECUTE FUNCTION update_updated_at_column();
```

#### DDL

```sql
-- ステータス用のenum型を定義
CREATE TYPE todo_status AS ENUM ('PENDING', 'COMPLETED');

-- テーブル定義
CREATE TABLE todo (
  id SERIAL PRIMARY KEY,
  title VARCHAR(32) NOT NULL,
  contents VARCHAR(256),
  status todo_status NOT NULL DEFAULT 'PENDING',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- updated_atを自動更新するトリガーを設定（PostgreSQLではON UPDATE句がないため）
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
   NEW.updated_at = CURRENT_TIMESTAMP;
   RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_update_timestamp
BEFORE UPDATE ON todo
FOR EACH ROW
EXECUTE FUNCTION update_updated_at_column();
```

### GraphQL スキーマ定義

- TodoStatus（ステータス列挙型）

```graphql
enum TodoStatus {
  PENDING     # 未完了（0）
  COMPLETED   # 完了（1）
}
```

- TODO型

```graphql
type Todo {
  id: Int!                 # 一意な識別子
  title: String!           # TODOのタイトル
  contents: String         # TODOの詳細内容
  status: TodoStatus!      # TODOのステータス（未完了/完了）
  createdAt: String!       # 作成日時
  updatedAt: String!       # 更新日時
}
```

- CreateTodoInput / UpdateTodoInput

```graphql
input CreateTodoInput {
  title: String!
  contents: String
}

input UpdateTodoInput {
  title: String     # 編集時に使用
  contents: String  # 編集時に使用
  status: TodoStatus # ステータス更新時に使用
}
```

※UpdateTodoInput では、更新対象に応じて任意の項目を指定できる。status のみ、または title・contents のみの更新にも対応する。

###  GraphQLクエリ / ミューテーション定義一覧

| No | 処理名   | 種別     | フィールド名   | 引数                             | 内容                                       |
|----|----------|----------|----------------|----------------------------------|--------------------------------------------|
| 1  | 一覧取得 | Query    | todos          | なし                             | TODO全件を取得する。ステータス昇順、更新日時降順で返す |
| 2  | 詳細取得 | Query    | todo           | id: Int!                         | 指定されたIDのTODOを1件取得する            |
| 3  | 登録     | Mutation | createTodo     | input: CreateTodoInput!         | 新しいTODOを1件登録する                    |
| 4  | 更新     | Mutation | updateTodo     | id: Int!, input: UpdateTodoInput! | 指定IDのTODOを更新する（部分更新）         |
| 5  | 削除     | Mutation | deleteTodo     | id: Int!                         | 指定IDのTODOを物理削除する                 |
